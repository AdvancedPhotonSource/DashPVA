Workbench Documentation

Overview and usage notes for the DashPVA Workbench viewer.

Introduction

The Workbench viewer provides 2D, 3D, and 1D visualization of HDF5 datasets along with tools for ROI and speckle analysis.

Features

  • Load single HDF5 files or entire folders
  • Visualize datasets in 2D and 3D (when PyVista is available)
  • ROI creation and statistics
  • ROI Calculator (A − B), ROI Plot, and ROI Math tools
  • Playback of 3D stacks with FPS control

Opening This Documentation

Use the menu Documentation → Open Documentation or press F1 within the Workbench window.

2D Viewer

Opening Files

Use the file button to open an HDF5 file or folder. Selectable datasets are highlighted in blue. Right-click any file in the tree to edit or inspect it.

ROI

Click Draw ROI to place an ROI on the image. The dock can be positioned on either side of the window.

Player

Use the frame slider or spinbox to scrub through 3D stacks. FPS playback is controlled from the player bar.

3D Viewer

Available when PyVista is installed. Dataset stacks can be rendered as volumes or isosurfaces.

ROI Basics

  • Draw ROIs via Draw ROI. Drag to move; handles to resize; rotate handle to rotate about the center.
  • Statistics (sum, min, max, mean, std, count, x, y, w, h) appear in the ROI dock. Right-click for: Show Stats, Rename, Set Active, Open ROI Plot, Delete, Save ROI.
  • The checkbox in the stats table toggles the ROI name label on the image.

ROI Sources

  • Memory — ROIs drawn in the current session.
  • File — ROIs from HDF5 metadata. File-based import is planned; calculations currently use in-memory ROIs only.

ROI Calculator Operations

  • Average Series: A − B — 2D mean-difference image mean(A) − mean(B) over frames, plus a per-frame series diff.
  • Pixel-wise: A − B — 2D image or 3D stack where each frame is A − B aligned frame-by-frame.
  • Custom Expression — Planned; currently disabled.

Matching & Alignment

  • Strict Match on — A and B must have identical shapes including frame count.
  • Strict Match off + Auto-Intersect on — Truncated/cropped to the common region: T = min(frames), H = min(height), W = min(width).
  • Strict Match off + Auto-Intersect off — Shapes must still match; otherwise aborted.

Time Axis and Include ROI Frames are present for future use and not currently applied.

Results

  • Series tab — Per-frame mean(A) − mean(B) plot with metrics table.
  • Image tab — Computed image or stack result.
  • Send to 2D Workspace — Pushes the result to the main viewer.
  • Save As HDF5 — Saves to /entry/data/result with attributes operation, roiA_name, roiB_name, strict, auto_intersect, notes.

ROI Plot

Open via right-click → Open ROI Plot. Two radio buttons at the top of the dock switch between modes.

Stack Mode

Plots a metric over every frame in the loaded stack. Both X and Y dropdowns are active — mix any two metrics, e.g. Sum vs Time or CoM X vs a motor angle. Options: Time, Sum, Min, Max, CoM X, CoM Y, and any motor position from the file. A frame line and slider stay in sync with the main viewer.

Single Frame Mode

Projects the current frame's ROI onto one axis — a 1D count profile. The X dropdown is hidden; pixel index is used automatically. Updates on every frame change or ROI move.

Projection equations

Let ROI(i, j) be the count at row i, column j in the ROI subarray of shape H × W.

Proj X — sub.sum(axis=0)
Px(j)  =  ∑i=0H−1 ROI(i, j)    j = 0 … W−1
Collapses rows → 1D array indexed by column (horizontal profile)
Proj Y — sub.sum(axis=1)
Py(i)  =  ∑j=0W−1 ROI(i, j)    i = 0 … H−1
Collapses columns → 1D array indexed by row (vertical profile)

Example use cases

  • Calculate the sum of fringes along an axis to measure fringe period or visibility
  • Inspect a diffraction peak profile along detector rows or columns
  • Check whether a beam or feature is centered within the ROI
  • Compare count distribution across one dimension before and after an experimental change

The ROI Math dock accepts expressions over flattened ROI data (x: indices, y: values, np: numpy) and renders multiple curves.

Saving ROIs to HDF5

Saved under /entry/data/rois/<name> with attributes x, y, w, h, source_path. Dataset is a 2D image or 3D stack depending on source.

Technical Details

  • extract_roi_stack(main_window, roi) — 3D source → (T, H, W); 2D → (H, W). Transform-aware via roi.getArrayRegion, fallback to pixel slicing. Returns np.float32.
  • align_stacks(A, B, strict, auto_intersect) — Normalizes 2D → (1, H, W). Strict: identical shapes required. Auto-intersect: crops to common (T, H, W). Returns (A_aligned, B_aligned, info).
  • per_frame_mean(stack) — 1D np.float32 of per-frame means, length T (length 1 for a single frame).

Tips & Notes

  • The Average Series plot value per frame equals the average of pixel-wise A − B within that frame.
  • ROI areas outside the image are filled with zeros for series computation.
  • File-based ROI selection and Custom Expression are planned and currently disabled.
